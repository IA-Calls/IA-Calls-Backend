==========================================================
ESTRUCTURA DEL BODY COMPLETO
==========================================================

El JSON se envía en el body de la petición POST con la siguiente estructura:

{
  "conversation_config": {
    "tts": {                    // Configuración de Text-to-Speech
      "voice_id": "string",     // ID de la voz (ej: "pNInz6obpgDQGcFmaJgB")
      "model_id": "string"      // Modelo TTS (ej: "eleven_turbo_v2_5")
    },
    "conversation": {           // Configuración de conversación
      "text_only": boolean      // false para audio, true para solo texto
    },
    "agent": {                  // Configuración del agente
      "language": "string",     // Código de idioma (ej: "es")
      "prompt": {               // Prompt del agente
        "prompt": "string"      // Texto del prompt
      },
      "initial_message": "string" // Mensaje inicial (opcional)
    }
  },
  "platform_settings": {},      // Configuración de plataforma (objeto vacío por defecto)
  "name": "string",            // Nombre del agente
  "tags": ["string"]           // Array de tags
}

==========================================================
NOTAS IMPORTANTES
==========================================================

1. El campo "conversation_config" contiene toda la configuración del agente
2. "tts.voice_id" es obligatorio - actualmente se usa "pNInz6obpgDQGcFmaJgB" (Adam)
3. "tts.model_id" es obligatorio para agentes no-inglés - se usa "eleven_turbo_v2_5"
4. "agent.language" establece el idioma del agente (actualmente "es" para español)
5. "agent.prompt.prompt" contiene las instrucciones del agente
6. "agent.initial_message" es opcional y solo se agrega cuando el grupo tiene firstMessage
7. Los "tags" ayudan a organizar y buscar agentes en ElevenLabs
8. El método createAgent() combina las opciones por defecto con las personalizadas
